# Stage-04 Charter — requirements-validation-and-concept-proof

## 1. Authoring Goal

- Target Stage Skill: `requirements-validation-and-concept-proof`
- Parent phase: product / requirements
- Goal: convert Stage-03 MVP and slicing assumptions into a validation-ready hypothesis loop, validation record, conclusion, and revision path that can support downstream design/architecture entry.

## 2. Problem This Stage Must Solve

This stage does not claim market truth by itself. It is responsible for answering:

- what exactly must be validated
- how validation should be run at low enough cost
- what evidence signals would support Go / No-Go / Revise
- how validation outcomes should revise earlier product-stage decisions

## 3. Upstream Inputs

- Stage-03 MVP definition
- Stage-03 slice explanation
- Stage-03 slice-map or equivalent structure evidence
- key assumptions to validate
- deferred items and rationale

## 4. Downstream Handoff

- Downstream stage: design / architecture
- Minimum handoff package:
  - validation conclusion
  - validation record
  - prototype or equivalent validation artifact (if needed)
  - revision recommendations
  - explicit unresolved risks if any remain

## 5. Gate Focus

- Has Stage-04 identified clear validation targets?
- Is there an explicit hypothesis → method → signal/threshold → result → decision chain?
- Does the stage produce a conclusion that can actually modify MVP/slice/priority thinking?

## 6. Diagram Focus

- Stage-04 diagram is recommended, not hard-required.
- If a `validation-flow` is present, it should make the learning loop explicit.
