# Placemaking：用户需要“你在这儿”来建立位置感

- 类型：原则
- 适用阶段：设计收敛
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（Navigation Systems / Placemaking）
- 摘要：在数字环境中缺少自然地标，必须显式提供位置线索（品牌/层级/当前位置）。

## 输入条件
- 用户从搜索/外链进入内部页面后无法判断“我在哪、属于哪一类”

## 关键步骤
1. 保证站点身份一致可见（名称/logo/视觉识别）。
2. 让层级结构可见并标出当前位置（breadcrumb/高亮/“You are here”）。
3. 在不同入口一致呈现（不要只在首页有线索）。

## 输出结果
- 更强的位置感与更少的迷路成本

## 适用边界
- 过度强调层级会限制横向探索，需要与 contextual nav 平衡。

## 常见错误
- 依赖浏览器 back/forward 代替系统自带位置线索

## 关联模板
- 位置线索清单（identity / hierarchy / current location）

## 下游使用建议
- 原型评审：随机挑内部页，3 秒内能说出“我在哪/能去哪”。
