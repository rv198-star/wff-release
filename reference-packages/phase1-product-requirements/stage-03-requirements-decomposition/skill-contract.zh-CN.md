# Stage-03 阶段契约（审计镜像）— requirements-decomposition-and-mvp-slicing

## 1. 这个文件是做什么的
- 这是 Stage-03 英文主契约文件的中文审计镜像。
- 它帮助人类评审者理解：Stage-03 到底解决什么、输入输出是什么、什么叫“最小可用体验闭环”、以及什么时候能交给 Stage-04。

## 2. 核心定义（对应英文主文件）
- Stage-03 的目标不是验证方案，也不是定架构，而是把 Stage-02 的全景结构变成可解释的 MVP 边界与切片逻辑。
- 它必须输出：
  - MVP 定义
  - 切片说明
  - 需求拆解结果
- 它必须明确：首批、后续、暂缓项，以及为什么这样切。

## 3. 核心边界
- 若没有上游全景结构，不能进入本阶段。
- 若切片只是“先做一部分功能”，但没有完整体验闭环和可解释的依赖/风险逻辑，就不算过关。
- 它可以带 review-bound 的 provisional 假设继续往下，但不能伪装成已验证事实。

## 4. 边界提醒
- 本文件不是操作流程。
- 本文件也不是最终交付模板。
- 它定义的是 Stage-03 的边界合同与准入准出逻辑。
