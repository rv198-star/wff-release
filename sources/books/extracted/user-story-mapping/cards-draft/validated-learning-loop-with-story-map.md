# 用故事地图支撑验证性学习循环

- 类型：方法
- 适用阶段：需求分析 / 需求验证 / MVP 规划
- 来源：`sources/books/original/用户故事地图.md`（第3章、第15章）
- 摘要：故事地图不只是需求组织工具，还可以作为验证性学习的结构骨架，把假设、原型、测试和反馈组织成可迭代修正的学习循环。

## 输入条件
- 当前产品方向中仍存在关键假设或不确定性
- 团队愿意通过小步实验与反馈来修正认知

## 关键步骤
1. 明确当前待验证的问题、假设或机会。
2. 用故事地图组织用户活动与候选解决路径。
3. 设计最小化试验或原型，验证关键假设。
4. 依据反馈调整地图、切片和优先级。
5. 重复循环，直到形成更可靠的认知。

## 输出结果
- 与学习目标绑定的需求/方案结构
- 更可验证的 MVP 和迭代切片
- 经反馈修正的需求理解

## 适用边界
- 适用于高不确定性、需要快速验证的产品探索场景
- 不适用于已被冻结的合规性文档整理类任务

## 常见错误
- 把故事地图当静态计划图，不允许因反馈调整
- 先决定解决方案，再寻找验证理由
- 把“学习”理解为主观讨论，而非可验证反馈循环

## 关联模板

- MVP 说明

## 下游使用建议
