# 搜索不是导航的替代品：别用 search 当 Band-Aid

- 类型：边界
- 适用阶段：设计收敛
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（Search Systems / Focus on more useful navigation systems）
- 摘要：搜索引擎常被用来掩盖糟糕导航。正确顺序是先修组织/导航，再上搜索并让两者协同。

## 输入条件
- 团队认为“加搜索就解决找不到问题”

## 关键步骤
1. 先审查导航与标签是否能支持浏览。
2. 只有在信息量与需求形态证明需要时再引入搜索。
3. 用 controlled vocab/metadata 让搜索与导航共享语言。

## 输出结果
- 更一致的 finding 体验（browse+search 协同）

## 适用边界
- 政治原因导致导航无法统一时，搜索可作止痛药，但要标注为权宜。

## 常见错误
- 默认配置装完就忘（导致不可理解结果）

## 关联模板
- Search readiness checklist（nav/metadata/ops）

## 下游使用建议
- 用于 Stage-2 决策：引入搜索必须给出“为什么导航不足以覆盖”。
