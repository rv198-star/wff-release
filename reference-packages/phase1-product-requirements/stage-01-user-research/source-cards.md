## 2. Required Method Assets
- direct user research posture
- fast user-group segmentation
- research execution and insight synthesis
- opportunity framing before solutioning
- explicit alternative comparison for first-wave user choice
- problem-mechanism framing, not just symptom listing
- evidence layering: observed fact vs interpretation vs inference

## 3. Optional Support Assets
- group psychology / opinion-leader perspective
- stakeholder analysis when multi-role decision chains matter
- additional market / competitor context when needed for opportunity framing

## 4. Boundary / Anti-Pattern Assets
- avoid generic persona labels without evidence
- avoid treating inferred content as confirmed user fact
- avoid turning Stage-01 into early solution design
- avoid using principle/governance assets as a substitute for concrete user understanding output

## 5. Bundle Roles
- `product-demand-fit`

- `behind-the-scenes-product`
  - user understanding, user grouping, product judgment strengthening
- `inspired`

- `lean-product-development`

## 6. Current Use Rule
- The common reasoning-kernel docs above are normative for how cards are turned into reasoning operations.
- This file only narrows which cards matter most for Stage-01.
- Use source bundles as controlled upstream assets.
- Do not copy card prose directly into Stage-01 runtime files.
- If a source gap blocks Stage-01 runtime drafting, record the gap first before proposing a new bridge artifact.
- At runtime, do not treat source bundles as passive background reading.
- Before claiming material activation, run the artifact through:
  - artifact-unit selection
  - method-card reasoning
  - evidence/uncertainty classification
  - maturity/confidence classification
  - bounded reasoning loop control
  - deepening/freeze decision
  - downstream handoff consequence check
- Before freeze, explicitly activate and apply:
  - one user-segmentation method asset
  - one evidence/insight synthesis method asset
  - one problem-framing method asset
- The reasoning evidence must make it visible which method families materially shaped:
  - chosen user boundary
  - final problem statement
  - need framing choice
- A card listed but not shown to change the conclusion counts only as nominal use, not material use.
