# 重构前必须先有可靠的自测试体系（快速反馈安全网）

- 类型：检查清单 / 方法
- 适用阶段：测试；开发实现
- 来源：`sources/books/original/phase-others/refactoring-improving-the-design-of-existing-code.md`（第1章 1.3；第4章主题）
- 摘要：重构依赖测试作为安全网。测试必须能自我检验并提供快速反馈；否则重构会退化成“边改边猜”，调试时间爆炸。

## 输入条件

- 你准备对已有代码做结构性调整（无论大或小）

## 关键步骤

1. 在动结构之前，确认关键路径有可重复运行的测试。
2. 测试必须能自我检验（失败时给出明确差异），避免人工对比输出。
3. 让测试运行足够快，以支持“频繁运行”。
4. 在重构过程中，以“每个小步骤后立刻运行测试”为节奏。

## 输出结果

- 重构的风险被显著降低；失败定位成本下降

## 适用边界

- 对遗留系统，短期内可能只能先建立 integration/acceptance safety net，再逐步补细粒度测试。

## 常见错误

- 没有测试就开始大改结构
- 测试不可重复、不可自检或太慢，导致重构过程无法形成反馈回路
