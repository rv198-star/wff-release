## v1.3.17 改了什么

v1.3.17 在 localized reader evidence 上取代 v1.3.16。

本版本主题是：

- canonical English P1/P2/P3/P4 artifacts 仍是 lifecycle source truth；
- localized reader translation 作为 non-blocking evidence lane 运行；
- 当配置 OpenAI-compatible LLM API 时，生成 P1 PRD、P2 ESP、P3 action cards 的 zh-CN reader artifacts；
- reader translation 不可用时，保留进度与 manifest evidence 语义，但不阻塞 canonical P1-P4 lifecycle；
- release install pack 默认携带锁定版本的 Mindthus runtime dependency，正常启动不再依赖首次运行时 GitHub clone。

## LLM API 边界

生成可供人工审核的 localized reader translation 需要 OpenAI-compatible LLM API。

翻译运行器读取 `config/generated-output-policy.json` 中的 `reader_translation` 配置和其中指定的 API key 环境变量。如果 key 写在本地 config 中，runner 会把它注入运行时环境，但不写入日志。

这个 API 不是 WFF 全局生命周期依赖。如果未配置，WFF 仍可运行正常 P1-P4。在无 API key 且 stdin 不可交互的环境中，reader preflight 会以 `no-api-key-noninteractive` 跳过 reader translation，而不是阻断 P1-P4。

## 操作耗时

R11 接受证据显示，完整 release-candidate 验证不是短 smoke：

- P1-P4 主流程：约 `8m49s`；
- 主流程后等待 full reader lane 完成：按 case 约 `66-79m`；
- 从启动到人工 Review 完成的人类观察窗口：约 `1h30m`。

如果 release 决策需要完整 reader evidence 和人工 Review，应预留至少 `90min`。
