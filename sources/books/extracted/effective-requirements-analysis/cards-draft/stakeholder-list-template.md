# 干系人列表模板

- 类型：模板
- 适用阶段：需求分析 / 项目启动
- 来源：`sources/books/original/有效需求分析（第2版）.md`（第4章，表4-1、表4-2）
- 摘要：关键干系人不是列得越多越好，而是要列得准。干系人列表通过类型、名称、说明、相关度、影响度五栏，帮助团队识别真正影响项目方向、验收与阻力的人。

## 输入条件
- 已完成目标/愿景的初步识别
- 需要识别会影响项目价值与验收的关键角色

## 关键步骤
1. 根据目标涉及的部门、分支机构和职责，初筛关键干系人。
2. 补充从风险角度识别的一票否决者、受负面影响群体、技术高风险团队等。
3. 逐条记录类型、名称、说明、相关度、影响度。
4. 将相关度、影响度作为内部判断信息，而非直接对客户公开。

## 输出结果
- 一张可用于调研计划、沟通计划和验收风险分析的干系人列表

## 适用边界
- 适用于多数中大型项目、B 端系统和多角色参与场景
- 对单一用户、低复杂项目可适当简化

## 常见错误
- 只看影响度，不看与项目的相关度
- 把所有人都列进去，失去“关键”意义
- 公开暴露内部打分，制造沟通风险
