# Stage-01 Charter — requirements-user-research

## 1. 本次 authoring 目标

- 目标 Stage Skill：`requirements-user-research`
- 所属大阶段：产品 / 需求
- 目标：把用户的初始意图、问题线索、机会线索收束成可供 Stage-02 直接消费的结构化输入。

## 2. 本 Stage 要解决的问题

本 Stage 不负责直接定义解决方案，而负责回答：

- 我们面对的是谁
- 他们处在什么问题/机会情境里
- 当前值得进一步结构化的机会是什么
- 哪些初步 User Case / User Story 可以作为下一阶段输入

## 3. 上游输入

- 用户原始描述 / 项目背景
- 业务机会描述
- 已有用户反馈 / 数据 / 访谈线索 / 市场信息（任一可用）

## 4. 下游 handoff

- 下游 Stage：`requirements-analysis`
- 下游需要的最小 handoff：
  - 用户群边界
  - 问题 / 机会清单
  - 至少一版 User Case / User Story 草案
  - 当前 assumptions / open questions

## 5. 当前最重要的 gate 焦点

- 是否已经从“零散输入”进入“结构化用户理解”
- 是否已经能支撑 Stage-02 建立需求全景结构
- 是否避免把 AI 推演内容静默当成已确认用户事实

## 6. 当前最重要的 diagram 焦点

- Stage-01 图示不是 hard gate
- 允许无图，但若无图必须有结构化表格替代（用户群 × 目标 × 痛点 × 证据来源）

## 7. 当前 authoring 范围边界

- 本轮只构建 Stage-01 的 authoring control artifacts
- 不在本文件中直接冻结最终正文
- 不在本轮重写 source bundles
