## 6. Current Use Rule
- The common reasoning-kernel docs above are normative for how cards are turned into reasoning operations.
- This file only narrows which cards matter most for Stage-04.
- Use source bundles as controlled upstream assets.
- Do not copy card prose directly into Stage-04 runtime files.
- If a source gap blocks Stage-04 drafting, record the gap first before proposing a new bridge artifact.
- Before freeze, run the current artifact unit through:
  - method-card reasoning
  - evidence / uncertainty classification
  - maturity / confidence classification
  - bounded reasoning loop control
  - deepening / freeze decision
  - downstream handoff consequence compilation
- At runtime, explicitly apply at least:
  - one assumption-targeting method asset
  - one validation-method comparison asset
  - one evidence-state honesty asset
- The stage must not freeze if the reasoning evidence cannot show how those method families shaped:
  - the exact validation target
  - the chosen method
  - the decision state
- A listed method family that does not change target selection, method choice, or decision consequence counts only as nominal use.
