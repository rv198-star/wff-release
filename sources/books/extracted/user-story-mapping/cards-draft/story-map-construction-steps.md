# 用户故事地图构建步骤

- 类型：步骤
- 适用阶段：需求结构化 / 需求拆解
- 来源：`sources/books/original/用户故事地图.md`（第5章）
- 摘要：故事地图的构建可以被简化为一套稳定步骤：先分步骤讲故事，再组织情节，探索替代路径，提取主干，最后围绕目标切分任务。

## 输入条件
- 团队已明确要讨论的业务目标、机会或用户任务范围
- 至少有一组参与者能共同口述用户活动过程

## 关键步骤
1. 分步骤写出用户故事或活动片段。
2. 组织情节顺序，形成故事流。
3. 探索替代故事和可选路径。
4. 提取地图主干，识别关键用户活动。
5. 按目标切分任务，用于 MVP 或迭代范围划定。

## 输出结果
- 一个初版故事地图
- 主干活动与细节任务的分层结构
- 为目标导向切片准备好的任务集合

## 适用边界
- 适用于需要通过协作梳理用户活动和交付边界的场景
- 如果只有静态制度型要求、没有明显用户行为流，这套步骤收益较低

## 常见错误
- 直接从任务细节开始，缺少主干
- 把地图画成未来系统结构图，而不是当前故事视图
- 不探索替代路径，导致地图僵化

## 关联模板
- 需求分析工作坊记录
- 故事地图草案
- MVP 切片说明

## 下游使用建议
- 可被“需求结构化 Skill”直接引用为执行步骤
- 可喂给未来的工作坊类 Skill 作为标准流程
- 可作为需求到设计 handoff 前的可选前置工件
