# 价值—频率双维优先级评估法

- 类型：方法
- 适用阶段：需求分析 / 迭代规划 / Backlog治理
- 来源：`sources/books/original/有效需求分析（第2版）.md`（第2章）
- 摘要：在 0～1 建设期，业务维通常是最合适的主评估维度。价值—频率双维评估法通过业务重要性、必备性、发生频率与未解决后果，给需求划分“必须做 / 应该做 / 可以做 / 可不做”。

## 输入条件
- 已完成需求还原与适度补充
- 需要对需求做业务维优先级判断

## 关键步骤
1. 明确需求发生在哪个业务场景中，并判断该场景是否属于关键业务。
2. 判断该需求是否为系统支持该场景所必备。
3. 评估问题发生频率（相对业务场景频率）。
4. 评估未解决后果，至少区分影响效益或影响效率。
5. 根据组合结果给出优先级，并纳入迭代筛选。

## 输出结果
- 需求优先级分层结果
- 可解释的 Backlog 排序依据

## 适用边界
- 适用于业务驱动的 B 端系统建设期
- 在成熟产品阶段，通常需要让位于用户维、竞争维或运营维

## 常见错误
- 让需求提出者自己填写优先级
- 四个维度全部加权平均，埋没关键需求
- 只讨论“重要不重要”，不讨论频率和后果
