# 开发实现阶段 Skill 包草案（v0）

## 文档目的

这份文档是给人类评审的 `v0` 大纲草案，用于说明开发实现阶段建议如何拆子阶段、每个子阶段包含哪些 Skill 结构、输入物、输出物、SOP 与引用素材。

---

## 先看这页：开发实现阶段总览

### 建议拆成 3 个子阶段 / Stage Skills

| 子阶段 | Stage Skill 名称（建议） | 目标 | 主要输入物 | 主要输出物 |
|---|---|---|---|---|
| 1 | 实现准备 / 开发计划对齐 | 把设计输入整理成开发可执行包 | 设计 handoff 包、接口契约、原型、约束清单 | 开发任务包、实现边界、技术实现准备说明 |
| 2 | 功能开发 / 约束实现 | 在既定边界内完成实现 | 开发任务包、接口与数据设计、测试前置约束 | 功能实现结果、实现说明、单测结果 |
| 3 | 实现自检 / 开发交接 | 确认实现满足下游测试与集成需要 | 功能实现结果、单测结果、约束清单 | 开发交接包、缺陷/风险清单、进入测试的说明 |

---

## 一、子阶段清单（评审用）

### 子阶段 1：实现准备 / 开发计划对齐

| 项目 | 内容 |
|---|---|
| 建议 Stage Skill 名称 | `implementation-readiness-and-task-alignment` |
| 主要目标 | 确保设计输入已转成开发可执行包 |
| 典型输入物 | 设计 handoff 包、接口契约、数据模型、交付型原型 |
| 典型输出物 | 开发任务包、实现边界说明、未决问题清单 |
| SOP 重点 | 核对输入、拆分实现任务、识别不确定项、补齐实现边界 |
| 需要的模板 | 开发任务包模板、实现边界模板、未决问题清单模板 |

### 子阶段 2：功能开发 / 约束实现

| 项目 | 内容 |
|---|---|
| 建议 Stage Skill 名称 | `constrained-feature-implementation` |
| 主要目标 | 在既定边界和契约内完成实现 |
| 典型输入物 | 开发任务包、接口契约、数据设计、约束说明 |
| 典型输出物 | 功能实现结果、实现说明、单元测试结果 |
| SOP 重点 | 按边界实现、保持契约一致、同步补充最小验证 |
| 需要的模板 | 实现说明模板、实现变更记录模板 |

### 子阶段 3：实现自检 / 开发交接

| 项目 | 内容 |
|---|---|
| 建议 Stage Skill 名称 | `implementation-self-check-and-handoff` |
| 主要目标 | 保证实现成果足以被测试和集成阶段消费 |
| 典型输入物 | 功能实现结果、单测结果、实现说明 |
| 典型输出物 | 开发交接包、风险/缺陷清单、进入测试说明 |
| SOP 重点 | 对照契约自检、确认边界未破坏、明确已知风险和交接方式 |
| 需要的模板 | 开发交接模板、风险清单模板 |

---

## 二、当前 v0 判断

- 开发实现阶段建议先保持 3 个子阶段，不必一开始细化到每种编码活动都独立成 Skill
- 当前文档与书籍素材对开发阶段的直接支撑较少，因此这个阶段目前更多是结构草案，不是可立即落地的正式阶段包
- 后续需要更多与实现、测试、交付直接相关的素材和模板支撑
