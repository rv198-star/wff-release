# 产品经理的双重核心职责：评估机会 + 定义产品

- 类型：边界
- 适用阶段：方法治理 / 产品探索 / 需求定义
- 来源：`sources/books/original/启示录_打造用户喜爱的产品.md`（第1章）
- 摘要：产品经理的核心工作不是写文档，也不是替销售转述需求，而是评估产品机会是否值得做，以及定义待开发产品应是什么样子。

## 输入条件
- 组织正在界定产品经理职责
- 当前项目存在“谁负责判断机会、谁负责定义产品”的模糊地带

## 关键步骤
1. 将“评估产品机会”与“定义待开发产品”识别为产品经理的双重核心职责。
2. 区分产品创意来源与产品定义责任，避免来源者即决策者。
3. 使用简化但有效的机会评估与产品定义方式，而不是堆砌文档。

## 输出结果
- 更清晰的产品经理职责边界
- 更稳定的机会评估与产品定义责任归属

## 适用边界
- 适用于软件产品团队、互联网产品团队、企业级产品团队
- 不适用于纯项目执行环境中不承担产品定义职责的角色说明

## 常见错误
- 把产品经理变成文档管理员
- 把产品机会评估外包给高层意见或销售输入
- 只定义低层需求，不承担产品机会判断

## 关联模板
- opportunity assessment
- product definition / spec
