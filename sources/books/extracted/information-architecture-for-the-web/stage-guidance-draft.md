# Information Architecture (IA) Stage Guidance Draft

## Goal

- 把《Information Architecture for the Web and Beyond》的方法资产转成 Phase-02 可引用素材：用 **组织/标签/导航/搜索/元数据** 等系统设计规则，提升 Stage-2 的“可找、可懂、可治理”。

## Best Stage-2 mapping

### architecture-definition-and-boundary-setting
- 用“top-down vs bottom-up vs invisible IA”明确：用户会深链进入系统，架构必须支持“中途入场”。
- 用组织系统（scheme/structure）定义信息宇宙边界与可扩展结构（taxonomy/hierarchy/hypertext/DB/tagging）。

### design-convergence-and-delivery-prototype
- 用标签系统规则控制歧义与术语（避免组织行话污染用户语言）。
- 用导航系统提供 placemaking（你在这/你能去哪/你能做啥）并平衡灵活性与杂乱。
- 用搜索系统决策规则避免“搜索当补丁”：先修导航与组织，再上搜索；并用 search logs 作为学习工具。
- 用 controlled vocabulary/metadata 作为“系统胶水”统一搜索/导航/标签。

## Common mistakes

- 把 IA 等同 sitemap/wireframe，把“找得到/看得懂”当作 UI 细节。
- 用组织结构图（org chart）或内部政治决定导航结构与标签。
- 先上搜索掩盖导航灾难（search as Band-Aid）。
- 忽视 deep-link：用户绕过首页进入内部页面后迷路。
