# Stage-02 输出模板（审计镜像）— requirements-analysis

## 1. 这个文件是做什么的
- 这是 Stage-02 英文输出模板的中文审计镜像。
- 它帮助人类快速核对：Stage-02 的交付物是否真正具备全景结构、关键约束、优先级分组和可交给 Stage-03 的 handoff 质量。

## 2. 最关键字段
- 目标
- 主干活动
- 任务结构
- 关键约束
- 初版优先级分组
- 高风险验证点
- provenance / confidence / verification

## 3. 图示要求
- Stage-02 图示是 hard gate。
- 至少要有 `story-map` 或 `requirements-structure` 之一。
- 最小元素包括：
  - 至少 3 个主干活动
  - 每个活动至少 2 个任务
  - 一条主干流程
  - 至少一个边界/排除项
  - 至少一个高风险待验证点

## 4. 交接要求
- 最终交给 Stage-03。
- handoff package 至少应包含：分析说明、结构视图、关键约束、优先级分组、高风险验证点、assumptions/open questions。
