## Reader-Lane Scope Clarification

In this release line, that scope is reader-lane-only: the tolerance change applies to localized reader translation structure / immutable-token checks and does not relax canonical P1-P4 mainline gates, phase verdicts, strict-runtime evidence, or claim ceilings.

## LLM API Boundary

Human-reviewable localized reader translation requires an OpenAI-compatible LLM API.

The reader translation runner reads `config/generated-output-policy.json` under `reader_translation` and the configured API key environment variable. If a key is stored in the local config, it is injected into the runtime environment without being logged.

This API is not a global WFF lifecycle dependency. If it is not configured, WFF can still run normal P1-P4. In a non-interactive environment with no API key, the reader preflight skips reader translation with `no-api-key-noninteractive` instead of blocking P1-P4.
