# WFF v1.4.2 Release Notes

发布日期：2026-06-01

状态：`released-with-claim-ceiling`

## 一句话

v1.4.2 是 v1.4.1 之后的维护稳定版：重点收口长技术文档的一致性控制、Path A / Path B 双路径断言控制、P1/P2/P3/P4/PX 关键生成链路的 claim-control sidecar、跨阶段污染检测、PhaseX 产物质量补强，以及 install pack 内容边界硬化。

它不是 v1.5，也不包含 #64 / #75 / #85 这类更大规模的后续重构或实验。

## 主要变化

### 1. 长文档一致性控制

本轮从 Issue #39 出发，修正了“流程串行、事实多源”的核心风险。Markdown、Mermaid、表格和 RTM 不再被视为机器事实源；长文档生成改为围绕 claim-control surface、rendered references、consistency audit 和 sidecar evidence 进行约束。

已覆盖的主线范围包括：

- P1 PRD；
- P2 Engineering Spec Pack；
- P3 Action Cards / implementation handoff surfaces；
- P4 validation closure surfaces；
- PX brownfield / existing-system reentry surfaces。

### 2. 双路径断言控制

v1.4.2 引入并落地 Path A / Path B 控制边界：

- Path A：简单生成物使用 local claim index，并在生成时登记 rendered/source claim refs；
- Path B：复杂长文档和多视图产物先编译 claim surface / writing plan，再生成正文、Mermaid、表格和 handoff 视图；
- 两条路径都要求新增事实通过 proposed claim / review-bound 机制进入，不允许从最终 Markdown 反向制造事实源。

### 3. 通用一致性审计与博弈测试

本轮把原先偏 PRD 的审计思路提升为更通用的 rendered-artifact consistency audit。它重点检查：

- rendered refs 是否能解析到已接受 claims；
- Mermaid / table / prose 是否出现未知或漂移的核心事实；
- section / block / view 层 coverage 是否满足计划；
- proposed claims 是否被错误当作已接受事实消费；
- 关键 ID、canonical names、aliases 和 relations 是否出现语义级不一致。

审计脚本仍是质量底线，不声明能证明“没有任何未知语义缺陷”。

### 4. PhaseX 质量与跨阶段防污染

PhaseX 四场景回流链路补强了产物质量评审、reentry 约束和跨阶段污染检测。PX / P1 / P2 / P3 / P4 之间现在有轻量 contamination boundary gates，用于防止示例项目、旧版本残留、上游场景文本或 release-case 内容泄漏到当前业务产物中。

本轮还强化了 install pack 内容边界：仓库内的发布案例与 proof snapshots 不再进入 skills install pack，只作为 authoring repository retained proof 保存。

### 5. 验证面治理

Issue #55 / #59 / #60 对验证脚本和证据面做了治理：区分当前门禁、诊断入口、历史复现和 retained proof，降低旧脚本被误当成当前 release gate 的风险。

彻底瘦身和删除归档不在本版本声明范围内，后续由 #64 和 #85 继续处理。
