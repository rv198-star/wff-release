# IA 交付生命周期：Research → Strategy → Design → Implementation → Administration

- 类型：过程
- 适用阶段：设计收敛
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（Getting Information Architecture Done / process of IA development）
- 摘要：IA 不是一次性产物，而是从研究到治理的连续过程；实施与管理决定可持续性。

## 输入条件
- 团队把 IA 当成“前期文档”，上线后无人维护

## 关键步骤
1. Research：理解 context/user/content（含政治与约束）。
2. Strategy：定义高层组织/导航结构与范围；给出候选内容类型与粗元数据。
3. Design：细化 sitemap/wireframe/metadata schema 与细节。
4. Implementation：标签/组织落地、测试、培训与文档。
5. Administration：持续评估、修剪、调整与治理。

## 输出结果
- 可持续的 IA 路线图与职责分配

## 适用边界
- 阶段边界在现实中模糊；关键是保持迭代与反馈回路。

## 常见错误
- 只做 Strategy/Design，不投资 Implementation/Admin
