# 3L5S Script Boundary

## Purpose

This file defines the WAE boundary for script assistance inside 3L5S.

Short rule:

> Scripts protect the shape, trace, and evidence surface of 3L5S. They do not replace the judgment inside 3L5S.

## What Scripts May Control

Scripts may control high-certainty workflow work:

- create a single-layer or three-layer working draft
- preserve the order of `Baseline -> Target -> Gap -> Strategy -> Breakdown`
- check for missing sections
- flag empty placeholders
- flag vague action verbs
- check whether action items mention evidence, acceptance, dependency, or next action
- record assumptions, unresolved questions, rejected alternatives, and loopback reasons
- produce repeatable reports for review

This is workflow control. It reduces mechanical omission.

## What Must Remain Agentic

Agentic judgment must own high-uncertainty truth:

- whether a signal is meaningful or incidental
- whether the evidence is strong enough to define a problem
- whether the problem statement is falsifiable in the actual domain
- which gap matters most
- which strategy is the best path under constraints
- whether surprising feedback requires looping back to Discovery or Definition

This is semantic control. It cannot be replaced by field completion.

## Evidence Bridge

Scripts should make evidence easier to inspect:

- require evidence paths or observable proof where claims are made
- surface claims that lack proof
- keep assumptions and unresolved questions visible
- cap confidence when evidence is missing

Evidence is not decoration. If evidence does not constrain a claim, it is not doing useful work.

## Naming The Output

Script output should be called:

> `3L5S Shape & Evidence Risk Report`

Do not call it:

- truth validation
- correctness check
- decision approval
- strategy certification

The report can say "this draft has missing evidence." It must not say "this strategy is correct."

## Regression Risks

Treat these as regressions:

- cleaner but thinner analysis
- more fields but less judgment
- every ambiguity forced into a confident answer
- domain language replaced by generic template language
- checklist completion treated as problem resolution
- script pass treated as proof that Discovery, Definition, or Resolution is complete

## One-Line Rule

> Use scripts to expose where 3L5S may be incomplete; use judgment to decide what the incompleteness means.
