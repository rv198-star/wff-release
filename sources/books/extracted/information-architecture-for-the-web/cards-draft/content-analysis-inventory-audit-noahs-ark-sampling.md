# 内容分析：Noah’s Ark 抽样 + inventory/audit 产出路线

- 类型：方法
- 适用阶段：架构定义
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（Content analysis / Noah’s Ark / content inventory & audit）
- 摘要：内容分析是 bottom-up IA 的基础。用“诺亚方舟抽样”覆盖主要内容物种，再产出内容清单与审计路线图。

## 输入条件
- 现有系统内容异质且规模大，top-down 愿景与实际内容脱节

## 关键步骤
1. Noah’s Ark 抽样：每种内容类型抓 2 个样本，覆盖 format/type/source/subject 等维度。
2. 内容清单（inventory）：列出内容在哪里、状态如何、缺口是什么。
3. 内容审计（audit）：给出迁移/改写/删除/补齐路线图。
4. 从内容结构与元数据中提取可用于 taxonomy/metadata schema 的模式。

## 输出结果
- 内容清单 + 审计结论 + 对组织/标签/搜索的输入

## 适用边界
- 过度科学抽样不一定更好；关键是多样性与可用洞察。

## 常见错误
- 只盘点数量，不提炼结构/元数据模式

## 关联模板
- 内容清单表（对象/类型/来源/owner/更新频率/元数据）

## 下游使用建议
- Stage-2：当团队争论“内容到底有什么”时，inventory 是止争工具。
