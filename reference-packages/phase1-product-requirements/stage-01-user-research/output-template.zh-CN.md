# Stage-01 输出模板（审计镜像）— requirements-user-research

## 1. 这个文件是做什么的
- 这是 Stage-01 英文输出模板的中文审计镜像。
- 它帮助人类评审者理解最终交付物长什么样、要填哪些字段、哪些地方必须显式标注 provisional 与来源。

## 2. 最关键字段
- 用户群边界
- User Case / User Story 初稿
- 结构化问题清单
- 结构化机会清单
- assumptions
- open_questions
- source / confidence / verification
- assumptions_to_validate

## 3. 图示要求
- Stage-01 没有硬性图示要求。
- 但如果不画图，必须用结构化表格替代，至少表达：用户群、目标、痛点、证据来源。

## 4. 交接要求
- 最终交给 Stage-02。
- handoff package 至少要包含：结构化调研摘要、用户群边界、User Case / User Story 草案、问题/机会清单、assumptions/open questions。
