## 6. Current Use Rule
- The common reasoning-kernel docs above are normative for how cards are turned into reasoning operations.
- This file only narrows which cards matter most for Stage-05.
- Use source bundles as controlled upstream assets.
- Do not copy card prose directly into Stage-05 runtime files.
- If a source gap blocks Stage-05 drafting, record the gap first before proposing prototype structure.
- Before freeze, run the current artifact unit through:
  - method-card reasoning
  - evidence / uncertainty classification
  - maturity / confidence classification
  - bounded reasoning loop control
  - deepening / freeze decision
  - downstream prototype handoff consequence compilation
- At runtime, explicitly apply at least:
  - one workflow-to-page-map method asset
  - one IA / page-grouping method asset
  - one object / state visibility method asset
  - one deferred-boundary preservation method asset
- The prototype-spec rationale is incomplete if those method families are not visibly reflected in:
  - chosen page map
  - route graph
  - page briefs
  - state coverage
  - deferred / non-goal boundary note
- A listed method family that does not change page, route, or state decisions counts only as nominal use.
