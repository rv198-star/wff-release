# 平台与姿态（Posture）应从使用情境推导，而不是从技术偏好决定

- 类型：决策规则
- 适用阶段：设计收敛
- 来源：`sources/books/original/phase-02-design-architecture/About-Face-4.md`（第9章 平台和姿态）
- 摘要：平台选择与姿态（用户注意力投入方式）决定交互密度与信息组织。先理解情境，再做平台/姿态决策。

## 输入条件
- 产品跨多设备/场景，交互密度与导航方式争议大

## 关键步骤
1. 明确用户场景：注意力、频率、环境约束（移动/桌面/公共设备等）。
2. 选择姿态：独占（sovereign）/暂时（transient）/后台（background）等，并据此设定界面规则。
3. 将技术平台决策尽量纳入交互设计成果（避免硬件先行锁死）。

## 输出结果
- 一组可解释的交互密度与导航结构决策

## 适用边界
- 同一产品可能包含多姿态子功能；需区分“整体姿态”与“局部工具姿态”。

## 常见错误
- 以视觉风格替代姿态决策（把 posture 当成 UI 皮肤问题）

## 关联模板
- 姿态选择记录（场景→姿态→界面规则）

## 下游使用建议
- 在原型阶段用姿态作为一致性检查：临时工具不应表现为独占主应用。
