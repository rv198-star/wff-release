# 探索和发现有用的价值原则

- 类型：概念
- 适用阶段：需求调研 / 需求分析 / 产品探索
- 来源：`sources/books/original/精益产品开发_原则_方法与实施.md`（第3章、第4章）
- 摘要：精益产品开发的第一性原则之一，不是预先假设价值已知，而是承认价值具有不确定性，并把“探索和发现有用的价值”嵌入产品开发与交付过程。

## 输入条件
- 当前产品方向、需求方向或业务假设存在明显不确定性
- 团队需要决定应先探索什么，而不是直接固化方案

## 关键步骤
1. 承认初始认知只是起点和假设，而不是已知真理。
2. 明确哪些用户、问题、渠道、方案和商业条件仍待验证。
3. 把探索与发现价值的活动纳入产品交付过程，而不是放在流程外侧。
4. 基于反馈和验证结果持续修正价值定义。

## 输出结果
- 更适合探索型产品工作的需求/方案判断框架
- 为验证型需求工作提供原则基础

## 适用边界
- 适用于市场、用户、业务模式、产品方向存在不确定性的场景
- 不适用于纯制度遵循型、无探索任务的静态规范整理工作

## 常见错误
- 把最初想法当成已经确认的价值
- 一开始就把价值定义固定死，不允许探索修正
- 将“交付速度”与“价值正确性”割裂开来
