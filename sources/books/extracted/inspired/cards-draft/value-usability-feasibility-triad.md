# 价值-可用性-可行性三元基本条件

- 类型：概念
- 适用阶段：需求分析 / 产品探索 / 架构设计前置判断
- 来源：`sources/books/original/启示录_打造用户喜爱的产品.md`（封面引导、前言、好产品靠设计）
- 摘要：好产品至少同时满足三个基本条件：价值、可用性、可行性。任何一项缺失，都意味着产品方案本身不完整，不能视作成熟定义。

## 输入条件
- 团队正在评估某个产品机会、功能方向或解决方案
- 当前需要判断产品定义是否足够成立

## 关键步骤
1. 检查该方向是否对用户和业务有真实价值。
2. 检查用户是否能理解并舒适地使用它。
3. 检查当前技术与组织条件下它是否可行。
4. 若三者中有一项缺失，则继续探索，而非直接进入开发。

## 输出结果
- 对产品定义成熟度的三维判断框架
- 为后续产品探索、设计和架构阶段提供共同判断语言

## 适用边界
- 适用于产品探索、需求定义、方案评估场景
- 不适用于只做纯技术重构、没有用户与业务价值判断需求的任务

## 常见错误
- 只验证价值，不验证可用性和可行性
- 把技术可行误当成产品成立
- 把可用性当成后置美化问题

## 关联模板
- 机会评估
- 产品探索记录
- 原型评审记录

## 下游使用建议
- 可被未来“需求分析 Skill”作为入口判断框架引用
- 可被“架构设计 Skill”用来检查方案是否在错误阶段被过早冻结
- 可作为 refusal 依据：若三元条件无法说明，不应宣称产品定义完成
