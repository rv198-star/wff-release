# 聚焦价值流动效率原则

- 类型：概念
- 适用阶段：架构治理 / 交付治理 / 流程设计
- 来源：`sources/books/original/精益产品开发_原则_方法与实施.md`（第2章）
- 摘要：精益产品开发的核心原则之一，是把关注点从内部资源效率转移到用户价值端到端流动效率；也就是关注价值是否顺畅流动，而不是单点资源是否始终忙碌。

## 输入条件
- 团队正在设计流程、协作模式、交付链路或多团队协作机制
- 已感受到局部效率高但整体交付慢、等待多、反馈迟的问题

## 关键步骤
1. 把用户需求或用户价值识别为流动单元，而非把资源忙碌程度当成核心目标。
2. 观察价值从提出到交付全过程的等待、阻塞和切换成本。
3. 优先优化端到端流动，而非局部资源利用率。
4. 在复杂产品中，通过前后拉通、左右对齐来提升整体流动效率。

## 输出结果
- 以价值流动为中心的交付治理视角
- 更适合后续设计流程、编排和看板系统的原则基线

## 适用边界
- 适用于多阶段、多角色、多团队协作的产品/项目环境
- 不适用于只需要静态归档、不存在流动优化目标的纯文档整理任务

## 常见错误
- 只看资源利用率，不看需求等待与切换
- 把局部最优误当成整体交付最优
- 只做实现阶段迭代，却不打通端到端价值流
