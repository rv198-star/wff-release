# 组织行话标签是用户敌意信号：只有内部人懂的词会毁掉转化

- 类型：风险
- 适用阶段：设计收敛
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（Labeling Systems / Starbucks labels / jargon）
- 摘要：用内部术语命名导航，会让绝大多数用户迷路并降低信任；这不仅是可用性问题，也是成本问题。

## 输入条件
- 导航标签反映组织内部结构、部门名或内部项目名

## 关键步骤
1. 识别“只有内部人懂”的标签（部门/流程/系统名）。
2. 用用户语言重写，并用内容示例验证代表性。
3. 在多终端（尤其移动端）重测标签语境，避免词义漂移。

## 输出结果
- 更清晰的路径选择与更高的信任

## 适用边界
- 内部门户可允许有限行话，但仍需面向新员工/跨部门用户。

## 常见错误
- 把“清晰（对内部）”误当“清晰（对用户）”

## 关联模板
- 标签重写对照表（internal → user-centric）

## 下游使用建议
- 作为 Stage-2 交付物 gate：若顶层标签含行话，必须给出用户语言替代与证据。
