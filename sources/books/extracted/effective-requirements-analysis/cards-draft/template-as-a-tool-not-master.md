# 模板是工具，不是主人

- 类型：风险
- 适用阶段：方法治理 / 需求分析
- 来源：`sources/books/original/有效需求分析（第2版）.md`（第6章）
- 摘要：任务板和输出物模板的目的是帮助分析者从关键角度思考，而不是要求每一栏都必须填写。若把模板当刚性清单强行填满，就会从“人使用模板”滑向“模板支配人”。

## 输入条件
- 团队准备使用本书提供的任务板、模板或输出物格式
- 当前项目存在裁剪需求或复杂度差异

## 关键步骤
1. 先明确当前任务要解决的核心问题，再决定使用哪些模板栏位。
2. 把模板当作思考提示器，而非格式合规表。
3. 对不适用的栏位允许留空、弱化或剪裁。
4. 保证最终保留的信息真正支持分析、决策与交接。

## 输出结果
- 更贴合上下文的需求分析产物
- 降低模板僵尸化和文档膨胀风险

## 适用边界
- 适用于所有模板化需求分析工作
- 不适用于受法规/合同约束必须完整填报的刚性表单

## 常见错误
- 机械地把每一栏都填满
- 让模板替代思考
- 为了模板完整牺牲信息质量与分析效率

## 关联模板
- 问题卡片模板
- 干系人列表模板
- 质量场景模板
