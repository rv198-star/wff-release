# 需求 / 产品阶段拆书素材聚合引用索引

## 这份文档的目的

这份文档不是融合包，也不是 Stage Skill 草案。

它的目的更轻，也更直接：

> 为后续 Stage Skill 建设提供一个高效的“素材选取入口”，帮助执行者快速知道：当前有哪些 source bundles、它们分别适合服务哪个阶段、哪类问题优先看哪一组素材。

因此，这份文档属于：

> 聚合引用层（aggregation-by-reference layer）

它保留以下原则：
- 不回写 source bundle
- 不复制 source card 正文
- 不提前把素材包合成 Skill
- 只做整理、归类、引用、索引

## 二、按主题组整理

- 主题定位：
  - 全景图

## 五、建议的使用姿势

### 1. 先用本索引选素材包，再进 source bundle
不要从几百张 cards 里盲搜，先确定：
- 你在做哪个阶段 Skill
- 你当前缺的是模板、方法、原则还是边界

### 2. 先按主题包取材，不要一开始就按单卡取材
主题包更能保留方法上下文，避免拿到一张卡却误解其边界。

### 3. source bundle 是参考资产，不是待改写草稿
后续使用中如果发现缺口，应优先：
- 在 Skill 设计层记录缺口
- 在未来定向抽取或 bridge 资产中补足

而不是立刻回写 source bundles。

---

## 六、当前不做的事

当前这层整理：
- 不把 source bundles 再融合成一个总包
- 不生成伪 Stage Skill
- 不要求所有 source cards 进入统一目录
- 不要求现在就做全面吸纳

它只负责一件事：

> 让后续真正写 Stage Skills 时，选材更快、更稳、更不容易失焦。

---

## 七、一句话结论

当前对现有拆书素材最合适的整理姿势不是继续融合，而是：

> 保留 source bundles 的主题独立性，在其上建立一层轻量聚合引用索引，作为后续 Stage Skill 选材入口。
