# Stage-01 阶段契约（审计镜像）— requirements-user-research

## 1. 这个文件是做什么的
- 这是 Stage-01 的英文主契约文件对应的中文审计镜像。
- 它帮助人类评审者理解：Stage-01 到底解决什么、输入输出是什么、哪些内容不能推演、什么时候不能过 gate。

## 2. 核心定义（对应英文主文件）
- Stage-01 的目标不是定义产品方案，而是把非正式意图收束成可供 Stage-02 使用的结构化用户理解输入。
- 必填输入至少包括：项目背景、业务机会描述、以及至少一条证据线索。
- 若业务机会描述缺失或研究对象不清，必须拒绝正式执行。

## 3. 推演边界
- 不能推演：目标用户核心边界、关键业务目标方向、真实硬约束。
- 可以 provisional 推演：proto-persona、User Case / User Story 初稿、问题/机会清单初稿。
- 所有 provisional 内容都必须经过用户 review，且必须标记来源、置信度、验证状态。

## 4. 输出要求
- 必须输出：
  - 用户群边界
  - 至少一版 User Case / User Story 草案
  - 结构化问题 / 机会清单
- 输出必须能直接供 Stage-02 使用。

## 5. 边界提醒
- 本文件不是操作流程。
- 本文件也不是最终产物模板。
- 它只定义 Stage-01 的边界合同。
