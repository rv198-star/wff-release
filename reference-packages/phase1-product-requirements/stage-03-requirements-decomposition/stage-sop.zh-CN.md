# Stage-03 操作说明（审计镜像）— requirements-decomposition-and-mvp-slicing

## 1. 这个文件是做什么的
- 这是 Stage-03 英文 SOP 的中文审计镜像。
- 它解释 Stage-03 如何从 Stage-02 的需求全景进入可解释的 MVP 切片，并把验证重点交给 Stage-04。

## 2. 流程主线
- 先确认 Stage-02 全景结构是否足够强。
- 再识别完整体验闭环。
- 再切出最小可用体验闭环。
- 再区分首批 / 后续 / 暂缓项。
- 最后解释为什么这样切，以及哪些假设应进入验证阶段。

## 3. 最大风险点
- 把“小一点的范围”误当成 MVP。
- 没有明确暂缓项。
- 没有依赖与验收目标，只是做阶段排期。
- 把还没验证的假设当成已确认事实。

## 4. 边界提醒
- 本文件不是边界合同，边界合同看 `skill-contract.md`。
- 本文件也不是最终交付版式，交付版式看 `output-template.md`。
