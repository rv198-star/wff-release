# 业务接口分析模板

- 类型：模板
- 适用阶段：需求分析 / 系统分解后细化
- 来源：`sources/books/original/有效需求分析（第2版）.md`（第8章，表8-1、表8-2）
- 摘要：业务接口分析不等于系统接口设计。它关注 Why（用途与业务价值）、What（交互过程）与约束，而不是技术实现 How，可作为子系统协作关系的需求层描述模板。

## 输入条件
- 已完成业务子系统划分并识别了服务关系
- 需要逐条明确子系统之间的接口需求

## 关键步骤
1. 填写接口概述：接口名称、提供子系统、使用子系统、业务目的、时机、频率。
2. 记录接口交互过程，必要时用时序图表达。
3. 用数据词典细化交互中的数据包结构。
4. 明确设计约束：协议要求、性能要求、环境要求及其他限制。

## 输出结果
- 一份面向需求层的业务接口分析记录

## 适用边界
- 适用于存在多个业务子系统的系统
- 单体、单子系统、无显著服务关系的系统可弱化

## 常见错误
- 直接跳入 API / 数据库 / 消息协议设计
- 只写接口名称，不写业务目的和使用频率
- 忽略交互约束与环境限制
