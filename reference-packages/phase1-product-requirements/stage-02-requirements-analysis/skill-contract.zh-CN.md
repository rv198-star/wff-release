# Stage-02 阶段契约（审计镜像）— requirements-analysis

## 1. 这个文件是做什么的
- 这是 Stage-02 英文主契约文件的中文审计镜像。
- 它帮助人类评审者理解：Stage-02 到底解决什么、输入输出是什么、哪些情况下必须退回 Stage-01、什么情况下可以 handoff 到 Stage-03。

## 2. 核心定义（对应英文主文件）
- Stage-02 的目标不是切 MVP，而是把 Stage-01 的用户理解结果转换成结构化需求视图。
- 它必须建立“全景结构”，而不是只有故事条目或任务清单。
- 它必须产出：
  - 需求分析说明
  - 全景结构 / 故事地图 / 等价结构视图
  - 关键约束清单
  - 初版优先级分组

## 3. 上游与下游边界
- 上游必须至少提供：用户边界、问题/机会清单、业务/产品目标方向。
- 若这些基础结构缺失，Stage-02 不能硬分析，必须退回 Stage-01。
- 下游交给 Stage-03 时，必须能支撑切片，而不是让下游重新拼全景图。

## 4. 推演边界
- Stage-02 可以 provisional 地推一个结构草案，但不能把 Stage-01 未确认的上游假设静默升级成确认事实。
- 若全景结构依赖未确认的核心边界，它只能作为 review-bound input 继续存在。

## 5. 边界提醒
- 本文件不是流程说明。
- 本文件也不是最终交付模板。
- 它定义的是 Stage-02 的边界合同与准入准出逻辑。
