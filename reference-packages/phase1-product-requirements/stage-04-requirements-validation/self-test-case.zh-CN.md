# Stage-04 自测案例（审计镜像）— 从 Stage-03 MVP 切片到验证结论

## 1. 测试目标

用一个有效的 Stage-03 MVP/切片包来验证 Stage-04 是否能够：

- 明确待验证对象
- 形成从假设到决策的验证链
- 产出可用的验证结论与修订建议
- 给设计 / 架构阶段形成可消费 handoff

---

## 2. 模拟上游输入

沿用 Stage-03 dry-run 输出，包括：

- MVP 定义
- 首批 / 后续 / 暂缓切片
- 待验证关键假设
- provisional 状态保留

---

## 3. 预期 Stage-04 行为

### 3.1 预期验证焦点
- 识别一个明确的待验证假设或关键决策点
- 选择低成本验证方式
- 形成会改变或确认前序判断的结论
- 给出修订建议或确认建议

### 3.2 预期 provisional 行为
- 不能把 Stage-03 的 review-bound 不确定性抹平。
- 如果证据仍弱，不确定性必须继续保留。

### 3.3 预期结构行为
- 必须让“假设 → 方法 → 结果 → 决策”链条清楚。
- `validation-flow` 推荐但不强制。

---

## 4. 预期最小输出结构

至少应包括：

- 验证对象 / 假设
- 验证方式
- 信号 / 反馈 / 结果
- 决策状态
- 修订建议
- design / architecture handoff package

---

## 5. 预期验收判断

### PASS 条件
- 有明确待验证对象
- 有可用结论
- 有修订建议
- 设计 / 架构阶段能知道什么被确认、什么仍有风险

### FAIL 条件
- 没有明确验证对象
- 有反馈但没有决策状态
- 有结论但没有修订后果
- 未确认假设被静默当成已验证事实
