# Stage-04 阶段契约（审计镜像）— requirements-validation-and-concept-proof

## 1. 这个文件是做什么的
- 这是 Stage-04 英文主契约文件的中文审计镜像。
- 它帮助人类评审者理解：Stage-04 到底验证什么、什么算验证结论、什么必须回灌前序阶段，以及什么时候能交给设计/架构。

## 2. 核心定义（对应英文主文件）
- Stage-04 的目标不是证明市场真相已经完全确定，而是把 Stage-03 的关键假设和关键决策点转成可解释的验证链与结论。
- 它必须输出：
  - 验证记录
  - 验证结论
  - 修订建议
  - （必要时）低保真原型或等价验证材料

## 3. 核心边界
- 若没有明确待验证对象，不能进入本阶段。
- 不能只有“用户反馈不错”这种模糊描述，必须有明确 decision state（Go / No-Go / Revise）。
- 不能有验证结论却没有回灌影响。

## 4. 边界提醒
- 本文件不是操作流程。
- 本文件也不是最终交付模板。
- 它定义的是 Stage-04 的边界合同与准入准出逻辑。
