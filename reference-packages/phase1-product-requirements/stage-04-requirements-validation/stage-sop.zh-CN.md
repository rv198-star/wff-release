# Stage-04 操作说明（审计镜像）— requirements-validation-and-concept-proof

## 1. 这个文件是做什么的
- 这是 Stage-04 英文 SOP 的中文审计镜像。
- 它解释 Stage-04 如何把 Stage-03 的 MVP/切片/关键假设，转成验证设计、验证记录、决策状态和修订建议。

## 2. 流程主线
- 先明确待验证假设 / 风险 / 关键决策点。
- 再选择低成本验证方式。
- 必要时做低保真原型。
- 再记录信号/反馈。
- 最后产出 Go / No-Go / Revise 与修订建议。

## 3. 最大风险点
- 没有明确假设就开始验证。
- 有原型但没有清晰验证对象。
- 有反馈但没有明确决策。
- 有结论但没有回灌建议。

- 本文件不是边界合同，边界合同看 `skill-contract.md`。
- 本文件也不是最终交付版式，交付版式看 `output-template.md`。
