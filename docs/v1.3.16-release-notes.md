## LLM API Boundary

Human-reviewable localized reader translation requires an OpenAI-compatible LLM API.

The reader translation runner reads:

- `OPENAI_API_KEY`;
- `OPENAI_BASE_URL`, unless `config/generated-output-policy.json` supplies `reader_translation.api_base_url`;
- `WFF_TRANSLATION_MODEL`, unless `config/generated-output-policy.json` supplies `reader_translation.model`.

This API is not a global WFF lifecycle dependency. If it is not configured, WFF can still run normal P1-P4. What cannot be claimed is API-generated localized reader translation evidence for human review.

When the translation runner is invoked without a working API, it reports `translation-failed` and exits non-zero. Release or review claims that depend on localized reader translation must then be marked as missing reader evidence / translation required. When a working API is provided, reader artifacts remain optional evidence and must still pass integrity / structure checks plus human review before quality claims are made.
