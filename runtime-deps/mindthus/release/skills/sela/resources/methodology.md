# 系统效率碾压局部优势（SELA）

> 顶层决策哲学原则 · 元层级 · 领域无关 · 优先级最高

## 原则

当一方在系统级费效比上形成数量级优势时，另一方的局部优势，即使真实、卓越，也无法阻止它被淘汰或边缘化。

局部优势不会消失，但会退守小众。

## 原型案例

- **中医 vs 现代医学**：资深中医对症下药的个案能力强，但培养慢、结果不确定、难规模化，长期被边缘化。
- **人工编码 vs AI Coding**：人工编码的可控性、可解释性、架构深度强，但单位产出效率差距拉大后，会逐渐让出主战场。
- **传统现做餐饮 vs 预制菜**：传统现做餐饮的风味与情感价值强，但成本高、稳定性差、难规模化，大众市场会被系统方案占领。

共同结构：

> 局部优势真实 + 系统性劣势致命 -> 长期被淘汰或边缘化。

## 判断时的三个追问

1. **对比是否公平**  
   是 `最好的 A` vs `平均的 B` 吗？决定领域命运的是平均 vs 平均。
2. **优势是否可规模化**  
   不可规模化的卓越，长期赢不了系统效率。
3. **劣势容忍度**  
   人对“可接受劣势”的容忍度远超直觉。`大家受不了` 几乎永远是错判高风险句。

## 单 Agent 多角色：防止效率原则变成偏执

SELA 的主要风险不是推理不够顺，而是太顺：一旦接受“系统效率碾压局部优势”，
同一个 agent 容易把所有局部价值都解释成迟早会被淘汰，滑向效率至上主义。

因此，非轻量 SELA 判断应先做单 Agent 多角色压力，而不默认拆成多个 Agent：

- `System Advocate`：强化系统效率、规模化、反馈速度、成本曲线、复利能力和长期主战场判断。
- `Local Defender`：为局部优势、信任、体验、不可替代场景、过渡窗口、伦理和不可逆风险辩护。
- `Timing Auditor`：只判断当前动作窗口，防止把“长期方向正确”直接写成“现在立刻全切”。

角色输出必须收束到当前动作，而不是扩展成完整价值哲学：

- `commit / trial / hold / wait`

如果 `Local Defender` 指出不可逆伤害、尊严底线、监管责任或关键过渡价值，
`System Advocate` 不能用效率主线直接覆盖它。若 `Timing Auditor` 判断行动窗口不足，
长期方向可以保留，但当前动作应降级为 `trial`、`hold` 或 `wait`。

不默认拆成多个 Agent。多个 Agent 只适合高影响、强分歧、长期方向或路径依赖很强的
判断，用来提高对抗烈度，而不是作为 SELA 的主路径。

## 时机检查：三问合一

`SELA` 判断的是长期系统方向，但长期方向不能自动推出当前动作。

时机检查不是独立原则，也不是补一套人文价值观或风险哲学。它只是让效率判断
在落地前多看一眼行动时机，避免把“长期会赢”误写成“现在立刻全切”。

三个问题合成一张轻量检查卡：

1. **是否需要临时保留？**
   如果新系统长期更优，但当前稳定性、成本、迁移条件或关键场景还没到位，旧方案
   可以作为过渡 lane 保留。保留不是否定趋势，只是承认当下切换成本。
2. **是否会锁死未来？**
   如果今天的结构、接口、流程、合同、数据形态或组织习惯会让以后迁移很贵，就不要
   因为长期方向清楚而过早重仓。优先选择可替换、可回滚、可试点的动作。
3. **是否处在合适行动窗口？**
   有些时候该马上切，有些时候该观察，有些时候该小规模试验。判断点不是口号，而是：
   现在行动能否比等待产生更多决策约束、成本优势或学习收益。

输出不要变复杂。只需要影响当前动作：

- `commit`：长期方向清楚，当前条件也成熟，可以主线推进。
- `trial`：方向有优势，但证据或迁移条件不足，先小试。
- `hold`：旧方案仍有临时价值，暂时保留，同时继续观察系统方案。
- `wait`：现在行动窗口不好，等待更便宜或证据更强。

## 取向

- 不用局部优势否定系统方向。
- 押注系统效率，理解但不眷恋局部优势。
- 主战场用系统方案，小众、高端或特殊场景保留卓越方案。

## 整体与局部

SELA 不是要求把所有局部都压平到同一个效率目标上。更准确地说，它要求整体给方向，局部给真实性，边界给不可逾越的约束。

真正可落地的不是一次性静态最优，而是可逆、可观察、可修正的持续逼近：

- `可逆`：错了能退
- `可观察`：能及时看见偏差
- `可修正`：反馈来了能改

因此，SELA 反对把“长期方向正确”误写成“现在立刻全切”。整体最优和局部最优的关系，不是消灭局部，而是让局部保留校准系统的能力。

## 不适用

- **不可逆伤害领域**：核心医疗决策、司法终审、核安全等。此时确定性优先于效率。
- **尊严、体验、伦理底线领域**：效率不是最高衡量标准。
- **过渡期内**：短期局部优势仍有市场价值；不要把长期趋势直接等同于当前切换动作，
  先做时机检查。

## 使用说明

### 调用时机

重大决策前，第一步扫一遍本顶层原则是否适用；不适用再进入下层具体判断。

### 正确用法

这是提醒式原则，不是结论式原则。

它的作用是：当我在为某个局部优势辩护时，检查这个辩护是不是情感驱动的。

检查完，结论可能仍然是保留局部优势。但那应该是清醒地保留，不是无意识地保留。

### 最常见的误用

把元原则当具体结论：

> 所以全部拥抱效率，抛弃传统。

这是机械套用，不是真正调用。

### 配套机制

- **记录**：每次用此原则做出的重大判断，记录下来。
- **复盘**：定期回看判断准确率。顶层原则必须用实战校准。
- **修订**：顶层原则也会进化，准备被修订，不是教条。

## 在决策哲学体系里的位置

这是顶层决策哲学原则之一，未来可陆续积累其他几条，形成完整判断底座：

- 【趋势判断】系统效率碾压局部优势
- 【证据权重】待补充
- 【风险态度】待补充
- 【时间尺度】待补充
- 【价值锚点】待补充

当前版本不急着补齐这些顶层原则。除非真实案例反复证明需要独立方法，否则保持
轻量：`SELA` 只增加时机检查，用来让效率判断落地得更全面，而不是扩展成完整
价值哲学。

顶层原则的价值在于彼此制衡。

单独一条会变成偏执。本条如果没有制衡，容易滑向效率至上主义。几条原则互相约束，才形成完整判断框架。
