# 模板僵尸反模式

- 类型：风险
- 适用阶段：需求分析 / 需求表达 / 工作坊协作
- 来源：`sources/books/original/用户故事地图.md`（第7章）
- 摘要：用户故事模板本应促进讨论，但一旦团队把模板当成替代思考和沟通的机械格式，模板就会异化为“模板僵尸”，导致故事看似规范、实则失去真实上下文。

## 输入条件
- 团队正在使用标准化用户故事模板
- 需求表达开始依赖格式化句式而不是共识构建

## 关键步骤
1. 识别模板是否仍在服务讨论，而不是取代讨论。
2. 检查是否存在“句式完整但上下文缺失”的故事。
3. 把模板重新降回辅助工具，把真实对话与场景澄清放回主位。

## 输出结果
- 对模板使用风险的识别
- 对需求表达方式的纠偏建议

## 适用边界
- 适用于故事模板、需求模板、轻量规范模板被机械化使用的场景
- 不适用于完全没有结构表达、仍处于纯口头混乱状态的早期探索

## 常见错误
- 以为模板字段填满就代表需求已清楚
- 用模板代替用户目标和上下文讨论
- 把“规范格式”误当成“高质量需求”

## 关联模板
- 用户故事模板
- 需求条目模板

## 下游使用建议
- 可被未来“需求分析 Skill”作为反模式检查项引用
- 可进入 review checklist，用于拦截格式正确但语义空洞的需求产物
- 可帮助定义 refusal 规则：若只有模板句式、无场景与目标说明，不进入下游阶段
