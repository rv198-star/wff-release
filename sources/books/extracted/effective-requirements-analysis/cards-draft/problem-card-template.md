# 问题卡片模板

- 类型：模板
- 适用阶段：产品探索 / 需求分析 / 需求规格整理
- 来源：`sources/books/original/有效需求分析（第2版）.md`（第3章，表3-1）
- 摘要：目标/愿景分析不应停留在空洞的定性目标，而应通过问题卡片把问题事实、业务后果、方案说明、预期结果与价值主张落成可交流的文档片段。

## 输入条件
- 已识别到一个清晰的问题场景或机会场景
- 需要将目标分析结果写入需求规格说明书或目标描述文档

## 关键步骤
1. 每张卡片只写一个问题场景或机会场景。
2. 填写问题描述与评估：事实、后果、频率、厌恶度、可替代性。
3. 填写方案说明：从黑盒或灰盒视角说明策略化方案。
4. 填写预期结果：系统投产后应达成的结果。
5. 视需要提炼价值主张，用“措施+效果”形成一句话目标。

## 输出结果
- 可直接进入目标描述章节的问题卡片
- 更具验证性的价值需求表达方式

## 适用边界
- 适用于目标不够清晰、需要场景化目标描述的项目
- 对法规驱动、目标已非常明确的项目可弱化使用

## 常见错误
- 一张卡片写多个问题场景
- 只写定性愿景，不写业务事实与后果
- 方案说明过于白盒化，客户高层无法理解
