# 关键质量属性列表与目标—场景—决策表

- 类型：模板
- 适用阶段：需求分析 / 非功能需求分析
- 来源：`sources/books/original/有效需求分析（第2版）.md`（第19章，表19-1、表19-3）
- 摘要：非功能需求不应停留在“高性能、高可靠”这类空洞形容词。应先识别并排序关键质量属性，再用目标—场景—决策表把质量需求落成具体威胁场景、技术对策与潜在风险。

## 输入条件
- 团队需要分析系统的安全性、可靠性、易用性、性能、可维护性、可移植性等非功能需求
- 已具备对业务环境、运行环境和用户特点的基本理解

## 关键步骤
1. 建立关键质量属性列表：质量目标类型、质量目标子类、说明。
2. 基于威胁影响度与出现频率，排序出关键质量属性。
3. 针对关键属性，逐条填写目标—场景—决策表：质量目标、具体场景、策略及风险。
4. 与开发团队共同评估不同质量对策之间的冲突与平衡。

## 输出结果
- 一份关键质量属性列表
- 一组可落地的质量场景/决策记录

## 适用边界
- 适用于存在明显质量风险、部署环境复杂或要求较高的系统
- 极简单工具类系统可大幅简化，但不应完全空白

## 常见错误
- 只写定性词，不写场景
- 为量化而量化，缺少历史数据支撑
- 使用全局化描述，忽略不同场景频率与差异
