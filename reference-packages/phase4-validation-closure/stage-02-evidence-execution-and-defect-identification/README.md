# Stage-02 Runtime Package — evidence-execution-and-defect-identification

## Purpose

This stage executes the planned validation work and converts outcomes into explicit evidence and defect records.

Its job is to produce reviewable execution results without:

- silently widening scope beyond Stage-01
- collapsing evidence requirements into narrative-only completion claims
- burying defects or blocked conditions inside summary prose

## Runtime package role

This directory holds the runtime-facing assets for Stage-02, including:

- `skill-contract.md`
- `stage-sop.md`
- `output-template.md`
- `source-cards.md`
- verification assets

## Upstream dependency

Stage-02 depends on a usable Stage-01 output package with explicit:

- acceptance checklist linkage
- coverage rationale linkage
- test entry/exit gate linkage
- execution-control linkage
- Stage-02 may-start declaration

## Downstream target

Stage-02 hands off to:

- `validation-closure-and-delivery-readiness-judgment`

with execution evidence, defect visibility, and known unresolved risks that are reviewable and auditable.
