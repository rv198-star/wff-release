# Stage-03 输出模板（审计镜像）— requirements-decomposition-and-mvp-slicing

## 1. 这个文件是做什么的
- 这是 Stage-03 英文输出模板的中文审计镜像。
- 它帮助人类快速核对：Stage-03 是否真的形成了 MVP 边界、切片逻辑、暂缓项，以及可供 Stage-04 验证的关键假设。

## 2. 最关键字段
- 完整体验闭环
- 最小可用体验闭环
- 首批切片
- 后续切片
- 暂缓项
- 切片依据（价值 / 风险 / 依赖）
- 待验证关键假设

## 3. 图示要求
- Stage-03 图示是 hard gate。
- 必须有 `slice-map`。
- 最小元素包括：
  - 至少 2 个切片
  - 每个切片都有能力边界
  - 每个切片都有验收目标
  - 每个切片都有关键依赖
  - 暂缓项明确可见

## 4. 交接要求
- 最终交给 Stage-04。
- handoff package 至少应包含：MVP 定义、切片说明、slice-map、待验证关键假设、暂缓项及其理由。
