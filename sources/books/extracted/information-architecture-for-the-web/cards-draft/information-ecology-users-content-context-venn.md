# 信息生态（Information ecology）：用户×内容×情境 三圈模型

- 类型：模型
- 适用阶段：架构定义
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（information ecology / three-circle Venn diagram）
- 摘要：信息环境是复杂自适应系统。用“用户、内容、情境”三圈模型避免只盯某一维度（只用户/只业务/只内容）。

## 输入条件
- IA 讨论陷入单维度：只做用户调研、只谈业务目标、或只盘点内容

## 关键步骤
1. 用户：信息需求、信息寻求行为、语言与能力。
2. 内容：类型、粒度、结构与元数据（现有与未来）。
3. 情境：业务目标/政治、预算与时间、渠道与技术基础设施。
4. 用三圈交集定位“策略重点”，并明确约束与 tradeoff。

## 输出结果
- 一份可解释的 IA research/strategy framing（知道该把手电筒照哪）

## 适用边界
- 三圈不是 checklist；它用来决定优先级与研究投入。

## 常见错误
- 把三圈当口号，不落到可验证的研究问题与交付物

## 关联模板
- 研究问题清单（按 user/content/context 分类）

## 下游使用建议
- 作为 Stage-2 kick-off 的 framing：先画三圈再定研究/策略交付。
