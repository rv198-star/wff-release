# wff-impl-action-cards SOP

1. Read the Phase-2 implementation entry and Engineering Spec Pack.
2. Resolve source authority from canonical P2 JSON bridges before markdown fallback.
3. Create ActionCards by component, operation, risk, and verification obligation.
4. Mark missing source authority as `review-bound` instead of filling by name inference.
5. Run the official ActionCard runtime.
6. Review the generated report before allowing downstream implementation.

Exit only when ActionCards expose what implementation and tests must prove.
