# 标签是“表示”：用最小页面/认知空间传达意义

- 类型：方法
- 适用阶段：设计收敛
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（Labeling Systems / labeling as representation）
- 摘要：标签用短词代表大块信息，让用户在不被信息淹没的情况下做出路径选择。

## 输入条件
- 导航/按钮/入口文字冗长或含糊，导致用户选择困难

## 关键步骤
1. 明确每个标签背后代表的“信息块/动作块”。
2. 追求高代表性（representational），减少歧义。
3. 需要解释时提供上下文（微文案/说明）而不是逼用户点进去。

## 输出结果
- 更高的可扫描性与可预测性

## 适用边界
- 图标标签在小屏重要，但仍需可理解的语义支持。

## 常见错误
- 把标签当品牌文案而不是导航语言

## 关联模板
- 标签评审表（代表什么/用户预期去哪/是否需要解释）

## 下游使用建议
- 在原型评审里做“标签预期测试”：用户说出点击后会到哪。
