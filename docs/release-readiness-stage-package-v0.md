# Phase-4 可选 Stage-04：上线前收口 / 发布准备扩展包草案（v0）

## 文档目的

这份文档是给人类评审的 `v0` 大纲草案，用于说明当项目需要正式发布门禁时，如何在 Phase-4 内启用可选 `Stage-04`，完成上线前收口 / 发布准备扩展，而不是再单独拆一个独立 Phase-5。

---

## 先看这页：Phase-4 可选 Stage-04 总览

### 建议拆成 2 个内部工作块 / Work Blocks

| 工作块 | 内部模块名（建议） | 目标 | 主要输入物 | 主要输出物 |
|---|---|---|---|---|
| 1 | 发布前收口 / 风险确认 | 确认是否具备发布条件、风险是否可接受 | 测试结论、风险清单、缺陷状态、交接信息 | 发布前收口结论、风险接受说明 |
| 2 | 发布准备 / 最终交接 | 形成最终交接包与发布准备包 | 收口结论、系统信息、交接清单 | 发布准备包、最终交接说明 |

---

## 一、内部工作块清单（评审用）

### 工作块 1：发布前收口 / 风险确认

| 项目 | 内容 |
|---|---|
| 建议内部模块名 | `pre-release-closure-and-risk-confirmation` |
| 主要目标 | 对发布条件和遗留风险做最终确认 |
| 典型输入物 | 测试结论、风险清单、缺陷状态 |
| 典型输出物 | 发布前收口结论、风险接受说明 |
| SOP 重点 | 检查发布条件、确认遗留问题是否可接受、形成正式收口判断 |
| 需要的模板 | 发布前检查清单、风险接受模板 |

### 工作块 2：发布准备 / 最终交接

| 项目 | 内容 |
|---|---|
| 建议内部模块名 | `release-readiness-and-final-handoff` |
| 主要目标 | 形成最终发布准备包和交接包 |
| 典型输入物 | 收口结论、系统信息、交接清单 |
| 典型输出物 | 发布准备包、最终交接说明 |
| SOP 重点 | 汇总交付信息、固化交接包、确认对外/对内准备状态 |
| 需要的模板 | 发布准备模板、最终交接模板 |

---

## 二、当前 v0 判断

- 这里不再建议把上线前收口单独建成 Phase-5，而是收编为 Phase-4 的可选 `Stage-04`
- `Stage-04` 本质上是“收口判断 + 最终交接”的扩展层，不建议在 v0 中过度细化
- 当前草案主要用于保证 Phase-4 在需要正式 release gate 时有明确扩展入口，后续细化依赖测试与发布实践资产
