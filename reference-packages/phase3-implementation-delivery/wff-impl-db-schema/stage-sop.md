# wff-impl-db-schema SOP

1. Read P2 data/storage contracts and database constraints.
2. Identify entities, tables, indexes, relationships, sensitive fields, and invariants.
3. Generate schema artifacts through the official runtime.
4. Preserve review-bound migration or data-retention unknowns.
5. Run schema-oriented verification when available.
6. Hand off schema outputs to backend and verification capabilities.
