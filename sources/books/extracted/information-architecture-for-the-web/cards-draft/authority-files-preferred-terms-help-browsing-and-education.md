# Authority file：首选词既帮助后台一致性，也帮助前台浏览与教育用户

- 类型：概念
- 适用阶段：设计收敛
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（Authority Files / preferred terms）
- 摘要：authority file 用首选词作为唯一标识，便于治理；前台呈现首选词能教育用户并保持索引简洁。

## 输入条件
- 同一概念有多种叫法，浏览索引/导航难以呈现

## 关键步骤
1. 为概念选择 preferred term 作为标识。
2. 收集 variants 映射到 preferred term。
3. 浏览/索引用 preferred term 展示，必要时做 term rotation（see/see also）。

## 输出结果
- 可治理的词汇系统 + 更一致的导航语言

## 适用边界
- preferred term 的选择需要治理机制（变更流程）。

## 常见错误
- 把所有变体都展示在前台，导致噪声

## 关联模板
- 术语治理流程（新增/合并/弃用）

## 下游使用建议
- 用于 Stage-2：跨团队内容平台中，把术语作为一等资产管理。
