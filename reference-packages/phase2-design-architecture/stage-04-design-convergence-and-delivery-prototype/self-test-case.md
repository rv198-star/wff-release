# Stage-04 Self-Test Case — design package convergence and delivery handoff

## 1. Test Goal

Use a complete Stage-01~03 brief to verify that Stage-04 can converge the design package into an implementation-facing handoff.

Focus:

- cross-stage consistency synthesis
- structural contradiction visibility
- unresolved-item visibility
- delivery-prototype-oriented expression
- all-scenario coverage carryover
- critical public-boundary sequence clarity
- technology selection evidence carryover
- acceptable-vs-optimal review quality
- readiness-claim calibration
- downstream consumption rules

---

## 2. Input Brief

> Stage-01 froze boundary and capability map. Stage-02 defined domain/module/service decomposition. Stage-03 defined data, storage, interface, and interaction package. Some latency, auditability, and long-term integration concerns remain unresolved.

---

## 3. Expected Minimum Output Shape
- architecture convergence summary
- prototype or equivalent delivery expression
- critical interaction sequence set
- optimality review
- design verification notes
- unresolved risk register
- implementation task sketch
- implementation-facing handoff package

---

## 4. PASS / FAIL

### PASS if:
- implementation phase can consume the package without re-synthesizing prior stages
- all known business scenarios remain covered even if only critical ones are diagrammed
- key public-boundary interactions are explicit without requiring internal class/method design
- major technology choices remain evidence-backed and reviewable
- final package explains why the chosen architecture beats the mainstream baseline under dominant constraints
- implementation task sketch stays above class/method/file-level freeze
- structural contradictions are surfaced or downgraded in readiness rather than hidden
- readiness wording does not exceed the verification/confidence/realizability evidence

### FAIL if:
- unresolved risks disappear
- delivery expression is confused with implementation completion
- structural contradictions are silently buried inside the final package
- key cross-boundary interaction order is left implicit
- internal private choreography is presented as mandatory
- technology choices lose their evidence trail during convergence
- convergence collapses stronger alternatives back into an unexplained mainstream default
- implementation task sketch turns into coding-level breakdown
- downstream usage rule is missing
- readiness claim is stronger than the evidence supports
