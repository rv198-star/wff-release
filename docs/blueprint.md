# 体系蓝图（草案）

## 目标
构建一组覆盖软件开发全生命周期的 Skills，并以“资料库 + SOP + 模板 + 阶段 Skill”方式组织，支持多人 / 多 Agent 协作。

## 四层结构

### 1. 知识底座层
- 拆书
- 文档抽取
- 知识卡片
- 术语表
- 反模式 / 风险案例

### 2. 方法治理层
- Skill 统一契约
- 阶段 SOP
- 输入 / 输出规范
- 验收规则
- 边界与流转约束

### 3. 阶段执行层
- 需求调研
- 需求分析
- 需求拆解
- 架构设计
- 模块设计
- 数据库设计
- 接口设计
- 功能开发
- 测试设计
- 测试执行

> 注：现阶段不要求一开始就全部独立成 Skill，可先按粗颗粒家族组织。

### 4. 编排协作层
- 阶段间交接
- 状态推进
- 依赖管理
- 看板映射
- 多 Agent 分工

## 当前 MVP 建议
先打通：
- 统一契约模板
- 知识卡结构
- 阶段 SOP 模板
- 需求调研 -> 需求分析 -> 需求拆解 -> 设计 主链

## 待讨论问题
- 是否先绑定一个样板项目类型
- 哪些阶段先合并为粗颗粒 Skill
- 模板规范与资料规范谁先冻结
