# 分类体系的广度定义了用户期望：你在定义“信息宇宙”

- 类型：决策规则
- 适用阶段：架构定义
- 来源：`sources/books/original/phase-02-design-architecture/Information_Architecture_For_The_Web.md`（Organization Systems / Topical organization schemes / breadth of coverage）
- 摘要：一旦定义了分类范围，用户会期待在该区域找到“现在与未来的所有相关内容”。

## 输入条件
- 分类命名/范围含糊，导致用户预期与实际内容不匹配

## 关键步骤
1. 对每个顶层分类明确其覆盖边界（包含/不包含）。
2. 用样例内容校验边界，避免“看起来合理但装不下”。
3. 为未来内容增长预留策略（扩展点/分面/标签）。

## 输出结果
- 可解释的 taxonomy boundary + 演进策略

## 适用边界
- 过宽会稀释意义，过窄会造成“内容跑到别处去”。

## 常见错误
- 用组织内部职责范围来定义分类边界（用户无感）

## 关联模板
- 分类条目说明（definition / inclusion / exclusion / examples）

## 下游使用建议
- 用于 Stage-2 输出物：taxonomy 需要 definition+examples，而不是只有名称。
