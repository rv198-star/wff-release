## 6. Current Use Rule
- The common reasoning-kernel docs above are normative for how cards are turned into reasoning operations.
- This file only narrows which cards matter most for Stage-02b.
- Use source bundles as controlled upstream assets.
- Do not copy card prose directly into Stage-02b runtime files.
- If a source gap blocks Stage-02b drafting, record the gap first before proposing a new bridge artifact.
- Before freeze, run the current artifact unit through:
  - method-card reasoning
  - evidence / uncertainty classification
  - maturity / confidence classification
  - bounded reasoning loop control
  - deepening / freeze decision
  - Stage-03 handoff consequence compilation
- At runtime, explicitly apply at least:
  - one NFR prioritization method asset
  - one conceptual domain-modeling method asset
  - one IA direction or subsystem-boundary method asset
  - one source-capability recompilation method that turns detailed structured source capability into a payload contract
  - one deferred-seam method that preserves future measurement / external extension points without falsely pulling them into MVP
- The specification rationale is incomplete if those method families are not visibly reflected in:
  - quality-attribute prioritization
  - domain model structure
  - module interface payload contract
  - deferred capability seam
  - Stage-03 slicing constraint explanation
- A listed method family that does not change the specification rationale counts only as nominal use.
