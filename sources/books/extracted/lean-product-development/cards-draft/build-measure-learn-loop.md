# 开发-测量-认知循环

- 类型：方法
- 适用阶段：需求验证 / 产品探索 / MVP 设计
- 来源：`sources/books/original/精益产品开发_原则_方法与实施.md`（第3章）
- 摘要：在高不确定性产品环境中，初始认知只是待验证假设。应通过开发-测量-认知循环，把价值探索和反馈验证嵌入产品交付过程，形成经实证修正的认知。

## 输入条件
- 存在待验证的产品假设、业务假设或方案假设
- 团队愿意通过小步实验与反馈持续修正理解

## 关键步骤
1. 明确当前待检验的概念、假设或方向。
2. 设计最小可行产品或最小试验来获取验证数据。
3. 基于反馈和测量结果验证或证伪原假设。
4. 更新认知，并进入下一轮循环。

## 输出结果
- 更接近实证的需求与产品判断
- 与验证目标绑定的 MVP / 试验设计

## 适用边界
- 适用于不确定性高、需要探索真实价值的产品场景
- 不适用于需求已冻结且仅需规范转写的情形

## 常见错误
- 把初始认知当成结论
- 开发了产品，却没有为验证设计测量环节
- 收集了数据，却不回到认知更新与后续规划
