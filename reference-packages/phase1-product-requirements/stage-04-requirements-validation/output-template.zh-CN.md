# Stage-04 输出模板（审计镜像）— requirements-validation-and-concept-proof

## 1. 这个文件是做什么的
- 这是 Stage-04 英文输出模板的中文审计镜像。
- 它帮助人类快速核对：Stage-04 是否真的形成了“假设 → 方法 → 信号/反馈 → 结论 → 修订建议”的完整验证链。

## 2. 最关键字段
- 假设 / 待验证对象
- 验证方式
- 原型或等价验证材料
- 反馈 / 信号
- 验证结论
- 决策状态（Go / No-Go / Revise）
- 修订建议

## 3. 图示要求
- Stage-04 图示不是 hard gate，但推荐 `validation-flow`。
- 若使用结构化表格替代，也应覆盖：
  - 假设
  - 方法
  - 信号/阈值
  - 结果
  - 决策

## 4. 交接要求
- 最终交给设计 / 架构阶段。
- handoff package 至少应包含：验证结论、验证记录、原型/等价验证材料、修订建议、剩余风险。
