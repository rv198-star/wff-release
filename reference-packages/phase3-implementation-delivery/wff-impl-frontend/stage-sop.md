# wff-impl-frontend SOP

1. Confirm frontend lane is requested or justified by upstream artifacts.
2. Read P1/P2 interaction and API surfaces.
3. Generate frontend through the official runtime.
4. Preserve UI assumptions, accessibility notes, and missing visual evidence.
5. Route unclear product/interaction truth upstream instead of inventing UI semantics.
6. Hand off UI evidence to P4 as review-bound when screenshots/manual proof are absent.
