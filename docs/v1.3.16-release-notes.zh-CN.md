## v1.3.16 改了什么

v1.3.16 继承 v1.3.15 RC 的 reader translation restart 内容，并在公开保留快照前完成 reader artifact boundary cleanup。

本版本主题是：

- release-packaged P1/P2/P3/P4 生命周期主线命令使用 canonical `--output-locale en`；
- 旧的确定性 `.zh-CN.md` audit mirror 不再是 P1 主线必需产物；
- localized reader translation 取代旧 mirror，成为供人工审核的本地化阅读表面；
- localized reader translation 仍是 optional evidence，不是 lifecycle blocker。

已知声明上限：P1 长文 PRD 仍会继承 source-native 中文输入和现有 P1 assembler helper text 的混合语言残留。v1.3.16 不声明 P1 canonical-English purity 已完全闭合。

保留快照中还包含一次后补 DeepSeek 重跑的 P1 PRD zh-CN reader artifacts。GEO / PetClinic P1 reader 均通过 immutable-token integrity 和 structure checks，人工 reader 分数为 `86 / 100` 和 `85 / 100`。这证明 P1 reader evidence 可用，但不证明 stable `90+` polished long-document localization。

## LLM API 边界

生成可供人工审核的 localized reader translation 需要 OpenAI-compatible LLM API。

翻译运行器读取：

- `OPENAI_API_KEY`；
- `OPENAI_BASE_URL`，除非 `config/generated-output-policy.json` 提供 `reader_translation.api_base_url`；
- `WFF_TRANSLATION_MODEL`，除非 `config/generated-output-policy.json` 提供 `reader_translation.model`。

这个 API 不是 WFF 全局生命周期依赖。如果未配置，WFF 仍可运行正常 P1-P4。不能声明的是 API 生成的、供人工审核的 localized reader translation evidence。

当翻译运行器在没有可用 API 的情况下被调用时，它会报告 `translation-failed` 并返回非零退出码。凡依赖 localized reader translation 的发布或评审声明，都必须标记为 missing reader evidence / translation required。提供可用 API 后，reader artifacts 仍只是 optional evidence，必须通过 integrity / structure checks 和人工 Review 后才能形成质量声明。

## 快照政策

v1.3.16 必须在 boundary cleanup 后保留 fresh GEO + PetClinic P1-P4 release-packaged strict-runtime 快照。该替代快照接受后，v1.3.15 RC 快照从 retained release snapshot storage 移除。

## 声明边界

v1.3.16 只证明开发 / 预生产生命周期内的生成与验证闭环。

它不声明真实线上 UAT、生产部署、生产发布批准、真实负责人签字、预算批准、上线授权、生产运维交接、生产风险承接、完整 P1-P4 reader translation coverage，也不声明 stable `90+` polished long-document localized-reader quality。
